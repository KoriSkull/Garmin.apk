# Инструкция: Сбор стрелок Яндекс.Карт для распознавания

## Цель
Собрать реальные изображения стрелок навигации из Яндекс.Карт и вычислить их хеши для использования в ArrowRecognition.

## Шаг 1: Подготовка приложения

1. ✅ Приложение уже собирается
2. После установки включите Accessibility Service:
   - Настройки → Специальные возможности → GHUD Lite → Включить

## Шаг 2: Настройка для Яндекс.Карт

1. Откройте GHUD Lite
2. Перейдите в "App Settings"
3. Убедитесь, что Яндекс.Карты добавлены и включены
4. Найдите поле для стрелки направления (DIRECTION_ARROW)
5. Запомните/запишите Resource ID

## Шаг 3: Сбор данных

1. **Запустите Яндекс.Карты**
2. **Постройте маршрут** с разными типами поворотов:
   - Прямо
   - Налево
   - Направо
   - Резко налево
   - Резко направо
   - Плавно налево/направо
   - Держаться левее/правее

3. **Начните навигацию** (можно симулировать движение)

4. **Дождитесь появления разных стрелок** на экране

## Шаг 4: Извлечение изображений

Приложение автоматически сохраняет все захваченные стрелки в:
```
/storage/emulated/0/Android/data/iMel9i.garminhud.lite/files/debug_arrows/
```

Формат имени файла:
```
arrow_20251206_195530_123_1234567890123456.png
       └─дата─┘ └время┘ мс └────хеш────┘
```

### Как получить файлы:

**Вариант A: Через ADB**
```powershell
adb pull /storage/emulated/0/Android/data/iMel9i.garminhud.lite/files/debug_arrows ./collected_arrows
```

**Вариант B: Через файловый менеджер**
1. Установите файловый менеджер с доступом к данным приложений
2. Перейдите в папку debug_arrows
3. Скопируйте файлы на SD-карту или отправьте себе

## Шаг 5: Анализ собранных стрелок

После сбора изображений:

1. Просмотрите файлы и сгруппируйте по типу стрелки
2. Запишите хеши для каждого типа:

```
STRAIGHT:    1234567890123456
LEFT:        2345678901234567
RIGHT:       3456789012345678
SHARP_LEFT:  4567890123456789
SHARP_RIGHT: 5678901234567890
EASY_LEFT:   6789012345678901
EASY_RIGHT:  7890123456789012
KEEP_LEFT:   8901234567890123
KEEP_RIGHT:  9012345678901234
```

## Шаг 6: Обновление кода

Обновите `ArrowDirection` в `ArrowRecognition.kt`:

```kotlin
enum class ArrowDirection(val value: Long, val hudCode: Int) {
    STRAIGHT(1234567890123456L, 0),
    LEFT(2345678901234567L, 1),
    EASY_LEFT(6789012345678901L, 2),
    EASY_RIGHT(7890123456789012L, 3),
    KEEP_LEFT(8901234567890123L, 4),
    KEEP_RIGHT(9012345678901234L, 5),
    RIGHT(3456789012345678L, 6),
    SHARP_LEFT(4567890123456789L, 7),
    SHARP_RIGHT(5678901234567890L, 8),
    NONE(0L, 0);
    // ...
}
```

## Шаг 7: Тестирование

1. Пересоберите приложение с новыми хешами
2. Установите
3. Проверьте распознавание во время навигации
4. Смотрите Debug Activity для проверки статуса

## Troubleshooting

### Стрелки не сохраняются
- Проверьте права доступа к хранилищу
- Убедитесь, что Accessibility Service включен
- Проверьте логи в Debug Activity

### Неправильный Resource ID
- Используйте UI Automator Viewer для поиска правильного ID
- Включите "Dump Mode" в Debug Activity для просмотра всех элементов UI

### Стрелки не распознаются
- Проверьте, что изображение квадратное
- Убедитесь, что размер примерно 132x132px (или кратный)
- Проверьте, что изображение бинаризуется корректно
