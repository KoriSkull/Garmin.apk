## Анализ изображений стрелок из Яндекс.Карт

### Текущее состояние

Я проанализировал декомпилированный APK Яндекс.Карт и обнаружил:

1. **Структура ресурсов**: 
   - В папке `res` находится более 500 PNG/WEBP файлов
   - Имена файлов обфусцированы (например: `-Ag.9.png`, `0GZ.png`, `NAv.webp`)
   - Нет явных имен типа "arrow_left", "maneuver_right" и т.д.

2. **Проблема**: 
   - Файлы имеют случайные/обфусцированные имена
   - Нужно визуально просмотреть изображения, чтобы найти стрелки навигации

### План действий

#### Вариант 1: Ручной поиск стрелок
1. Открыть папку `C:\Users\mts88\Documents\GHUD\ghud-lite\src\ru.yandex.yandexmaps_738726680_rs\res`
2. Просмотреть PNG/WEBP файлы в проводнике Windows (режим "Крупные значки")
3. Найти изображения стрелок направления
4. Скопировать найденные файлы в отдельную папку

#### Вариант 2: Анализ через приложение
1. Запустить Яндекс.Карты на устройстве
2. Включить навигацию
3. Использовать Android Studio Layout Inspector или similar tool
4. Найти ID ресурса стрелки в UI
5. Сопоставить ID с файлом через `resources.arsc`

#### Вариант 3: Создать Python скрипт для анализа
Создать скрипт, который:
- Загружает все PNG/WEBP из папки res
- Анализирует размер и форму изображений

- Создает HTML-галерею для визуального просмотра

### Рекомендация

**Самый быстрый способ** - использовать Вариант 1:
1. Откройте папку в проводнике Windows
2. Установите вид "Огромные значки" или "Крупные значки"
3. Прокрутите и найдите изображения стрелок
4. Они должны быть зелеными/белыми стрелками на прозрачном фоне
